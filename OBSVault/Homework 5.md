## Objective

Testing the initial version of the app, specifically how easily can a person that has never seen the app before navigate and use the app and perform the most basic of functions the app can provide.

## Methodology

All tests were performed over a video call, where the participants shared their phone screen and performed the required tasks. The participants were the same participants that were used for hypothesis testing as the participants then requested to be kept in touch with the project. As such long winded introductions and explanations were skipped. Timing was also not done as most tasks featured an some part where the user could spend an arbitrary amount of time which doesn't reflect actual performance of the application. The participants had a choice of either visiting the web demo version of the application or downloading the apk from github and installing it on the device. No disclosures or NDAs were necessary due to the app's source code being openly available and licensed under AGPL-3.0

## Product version

Version 0.2.0 of the app was used (git tag v0.2.0) which was available at the time as both an downloadable apk and a web demo.

## Participants

All 3 participants were students. As the primary audience of the application is student working part time "gigs" to pay their bills. The participants of the tests were a subset of the participants of hypothesis testing session as said participants asked to be kept in the loop about the project.

## Design

Surveys and big explanations were as mentioned before skipped as the participants were familiar with the project, however it must be noted that the participants were seeing a demo of the app for the first time, they previously only participated in interviews were only the concept was discussed.

Participants were tested in core tasks. Of starting and stopping a timer, editing a time entry, as well as the initial setup of creating a profile of the times. No measuring of time happened, due to the tasks having arbitrary length (the users could stop the timer when they wanted or take as long as they wanted to name a profile)

After each task as well as in the end the participants were asked if they had any inputs or comments. If they had any those were discussed further with them.

# Findings

The participants blazed through the tasks, sometimes going getting ahead of test conductor's instructions. Some questions were asked about what certain UI elements do, but the questions were also phrased in a way so the users were asking for confirmation about functionality, and not from being to unable to understand what the UI element did. One bug was discovered with the 3rd tester which after further examination only happened on the demo web version, however the bug was absent from the apk version.

One user appreciated that the most recent timed entry for a profile appeared on the main timing screen for said profile. Most users also noted (after "flash banging themselves") that the theme switcher should go in the setting somewhere safe from an accidental press.
